SCA tools


Â 

This beta release provides a preview of SCA Tools for the
development of OASIS SCA v1.1 applications, including deployment to WebSphere
Application Developer V8.5 Beta. For more information about OASIS SCA 1.1, see http://oasis-opencsa.org/sca.

Using the version of IBM Installation Manager provided with this
beta release, you can select new installation options for SCA Tools that enable
you to install either one or both of the existing OSOA SCA 1.00 or new OASIS
SCA 1.1 Tools. The OSOA and OASIS SCA tools can be installed and work
independent from each other.

If you developed SCA applications with an earlier version of
Rational Application Developer, then you will find the new OASIS SCA tools to
be very similar to the existing OSOA SCA tools. You can use wizards, editors
and validators to author the various SCA artifacts – composite, contribution –
that conform to the OASIS SCA 1.1 assembly model. In this beta release, you can
use the following SCA extension types:

  • Composite, and Java
    Implementation types
  • Default (SCA) and Web service
    bindings
  • JAXB 2.0 data bindings

The SCA tools work closely with the Java (JDT) and Web
Services tools, enabling you to locate the various interface, binding or
implementation artifacts (Java interfaces, classes, WSDL port types), or make
use of the new OASIS SCA annotations.

Once you’ve developed your OASIS SCA 1.1 application you can deploy
it to the new OASIS SCA 1.1 runtime provided by WebSphere Application Developer V8.5 Beta (March Refresh) and test it.
