New and Noteworthy in the WebSphere Application Server, WebSphere Developer Tools and Rational Application Developer Beta Refresh

The beta refresh introduces changes for WebSphere Application Server V8.5.Next, WebSphere Application Server Developer Tools for Eclipse V9.0 and Rational Application Development for WebSphere V9.0. These changes include a new installation model, functionality that shrinks your liberty environment as well as many other notable features, for example support for WMQ and Interoperability between WebSphere Application Server full profile and the Liberty profile.

Packaging

New Installation Model

The beta refresh introduces a new modular installation model. This enables you to install a small runtime environment consisting of web profile elements, such as EJB, CDI, Servlets etc. Then if additional features such as, WMQ or MongoDB are required they can be added by installing the extended installation on top.

Minify

With the use of Minify you can now shrink your Liberty runtime environment. Minify removes features that are not being used by your application, which provides you with a lightweight, tailored deployment, suited to the requirements of your application.

Messaging

Support for WMQ

WebSphere MQ is now supported as a message provided in the Liberty Profile. You can configure WebSphere MQ resources for applications (for example queue connection factories) and manage messages and subscriptions that are associated with JMS destinations in the Liberty profile. You can administer security through WebSphere MQ. Additional configuration might be required in WebSphere MQ depending on the resources that are used by your application.

MDB Supports Topic Space

The Beta introduced the support for MDB with p2p (Point to Point) messaging; the beta refresh introduces the support for Topic Space (Publish Subscribe) with MDBs.

JMS Security Model

You can now secure JMS communication between two Liberty servers by enabling the ssl-1.0 feature and configuring ssl on both the server and the client.

Interoperability between full profile and Liberty profile

The beta refresh introduces support for enabling Interoperability between the Liberty profile and WebSphere Application Server full profile.

The JMS applications that are deployed within the Liberty profile can connect to the messaging engine that is running within the WebSphereÂ® Application Server full profile (that is, Service Integration Bus). Similarly, the JMS applications that are deployed within the WebSphere Application Server full profile can connect to the messaging engine that is running in the Liberty profile.

Security

EJB Security

The run-as-mode is now supported in the IBM extensions file ibm-ejb-jar-ext.xml, all security related annotations and the deployment descriptor file ejb-jar.xml. For example, you could specify the following in your ibm-ejb-jar-ext.xml file so that the callers of the myMethod method will run as the user mapped to the Manager role.

appSecurity-2.0 feature

The new appSecurity version (appSecurity-2.0) supersedes the older version (appSecurity-1.0). The appSecurity-2.0 feature does not include LDAP user registry and no longer automatically includes the servlet-3.0 feature. For example, you should make the following changes to your server.xml.

appSecurity-2.0

servlet-3.0

TrustAssociationInterceptor

You can now develop a custom TAI as a third party feature by implementing the com.ibm.wsspi.security.tai.TrustAssociationInterceptor interface provided in the Liberty profile server and creating a product extension.

Password hashing

The basic registry password can now be stored using a one-way hash, you are able to use the securityUtility command to hash your password, for example:

./securityUtility encode –encoding=hash myPassword
{hash}ATAAAAAIYv4jBERcoC1AAAAAIEI2cbitb6frXRDGjzbjAeADjgxXPs1guj2Tgj3D3GEP


Federated repositories with LDAP repositories

With LDAP repository federation support, two or more LDAP repositories can be configured in the server.xml file, and you can get the consolidated results from multiple repositories for all LDAP operations.

Additional WS-Security X.509 template added for Web Services.

Administration

Stack traces printed to console are now automatically trimmed to remove frames for server internal code.

Clustering

Static cluster management

Static clustering allows for servers to opt-in to a cluster which is statically defined. The servers in a cluster can be started, stopped and queried as a group.

Collectives

Collectives are the new topology used to build out an administrative domain for a set of servers. A collective consists of at least one collective controller and typically some number of members.

Web Services

EJBs as a JAX-RS Resource

In this beta refresh of Liberty, you can expose a RESTful interface to the enterprise bean using Java API for RESTful Web Services (JAX-RS). By implementing JAX-RS annotated enterprise beans, you keep the EJB functionality including transaction support, injection of Java EE components and resources, and other EJB session bean capabilities.

CDI in JAX-RS Applications

JAX-RS classes in a Contexts and Dependency Injection (CDI) enabled archive can have @javax.inject.Inject annotated injection points, participate in JCDI lifecycle scopes, raise JCDI events, and have method interceptors and decorators.

You can now support creating EJB JAX-WS Web Services targeting Liberty. A new deployment descriptor editor is now available for custom binding files for Liberty

File transfer

The beta refresh introduces two new items for file transfer:

  • It can now be used together with the RoutingContextMBean, allowing files to be transfered from/to a client to/from a collective member through a proxy collective controller
  • It has also been enchanced so that it’s upload operation now allows users to specify if they want to expand an upload archive automatically

SPI and Features

Product Extensions

A Product extension can now be defined to liberty by putting a properties file in wlpetcextensions. The properties file specifies the location of the product extensions jars and manifest files.

Manifests

Features listed under IBM-Provision-Capability need to use the name of the feature specified on Subsystem-SymbolicName.

under Subsystem-Content features included with type=osgi.subsystem.feature need to use the name of the feature specified on Subsystem-SymbolicName.

A feature from a product extension is specified in server.xml by prefixing the feature name with the name of the product extension.

productA.properties in wlpetcextensions with a feature named feature1 would be specified in server.xml like this productA:feature1-1.0

Auto features

Auto features are specified in the manifest file under IBM-Provision-Capability.

Command-line utilities

A new option has been introduced: –productExtension=. If the user does not specify this option then the Liberty core features are outputted. If the user specifies –productExtension=usr, then the product extension in the default user location is outputted. If the user specifies –productExtension=, then the features for the product under that name/installation (otherExtProd.properties) are outputted.

WAB Security in User Features

User features that contain servlets (WABs) can now be protected with authentication and authorization. To secure a user feature WAB include the com.ibm.wsspi.appserver.webBundleSecurity-1.0 feature in the features manifest file. Use the authorization-roles element in the server.xml to map your WAB’s JEE roles to users, groups and special subjects.

CDI tools

New editor for beans.xml deployment descriptor editor with in-editor validations

Dynacache tools

Create and edit configuration files for WebSphere’s Dynacache.

Maven tools

Expanded support for Maven/OSGi projects with EJB and JPA content

SCA tools

The tool now detects when an SCA 1.1 composite references artifacts external to
the SCA project. You have the option to add a project reference or Java reference to ensure proper
compilation, export and deployment. (This ability is already supported for SCA 1.0 projects)

You can now test your SCA services that are configured with Web Service bindings using the Generic Service Client (GSC).

Portal tools

Support for creation of jQuery based Portlet projects is enabled through portlet project wizard to allow easy usage of jQuery APIs using the content assist option, drag and drop of jQuery mobile widgets from palette and visualization of jQuery Mobile widgets in RPE. Additional features include auto-generated initialization files and javascript code, and ability to use implicit portlet variables in portlets jsp. A customer will however need to procure their own jQuery library.

Code Coverage tools

Update Code Coverage launch history preference to set a higher limit for delete to be triggered and allow the option of not deleting launches.

IWD Tech Preview Refresh

Virtual Application Pattern Type

Where you select the Liberty Profile pattern has now changed from Web Application Pattern Type 2.0 to WebSphere Application Server Liberty Pattern Type 8.5.

Dynamic scaling policy

Adding a dynamic scaling policy for either CPU or Web Response scaling, will enable your web application to create/deconstruct multiple identical server instances dependent on the threshold set on the policy.

Connect in component

Adding the Connect In component and attaching it to your Liberty Profile will enable incoming communication via a specified port number.

Connect out component

Adding the Connect out component and attaching it to your Liberty Profile will enable outgoing communication via a specified port number.

Archive component

Adding a Archive component allows you to specify a compressed file (.zip) and a destination directory to which the compressed file is to be extracted.

SSL Endpoint

Https is now supported for web applications.
