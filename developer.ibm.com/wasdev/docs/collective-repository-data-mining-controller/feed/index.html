]]> By: Michael C Thompson https://developer.ibm.com/wasdev/docs/collective-repository-data-mining-controller/#comment-142460 Wed, 20 Jan 2016 17:43:32 +0000 https://www.ibmdw.net/wasdev/?post_type=doc&p=4650#comment-142460 The data is persisted to disk in the collective controller’s server configuration: ${server.output.dir}/resources/collective/
Because the repository has a simple CRUD model and follows a path-based naming pattern, it should be easy to store and read application data into the repository.

Both options seem reasonable, it really depends on what is the easiest for you to implement and wether or not your app would benefit from a shared data model like the collective. Of course, you can always use a database too!

Discovering the web apps is easy, there is a REST API on the controller that can be used to discover apps in the collective. controller host:port/ibm/api/collective
