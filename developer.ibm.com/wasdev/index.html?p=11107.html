Announcing: New Liberty features and tools


Laura Cowen
Published on December 9, 2014 / Updated on May 13, 2016
1 Comments

Ooo new shiny!

Yup, some of the cool Java EE 7 features that you’ll have seen in recent betas are now available in WebSphere Application Server Liberty Profile V8.5.5.

There’s also a tech preview of the new Migration Toolkit for Application Binaries and updates to the existing tech preview of the Migration Toolkit for Liberty.

So what are the shiny new Liberty features?

  • Java Servlet 3.1 The new servlet-3.1 feature provides an implementation of JSR 340 Java Servlet 3.1. The feature includes:
    • Non-blocking I/O – Increase scalability by freeing up server resources when waiting on a client to send or consume data.
    • Upgrade processing – Increase efficiency and functionality of client communication using an application defined communication protocol.
    • Large data – Send and receive more than 2GB of data.
  • Java API for WebSocket 1.0 Provides JSR 356 WebSocket support for web applications.
    • WebSockets can replace HTTP polling and streaming algorithms to provide faster and more efficient communication between a user’s browser, or other WebSocket client application, and a web application running on a Liberty server.
    • WebSockets provides a full-duplex, long-lived communication link between client and server endpoints, enabling you to easily manage the reading and writing of data and the lifecycle of web application connections.
    • WebSocket client endpoints can be used from within a web application running on a Liberty server to communicate to other WebSocket server endpoints. This allows for both efficient application development and run-time communication.
  • Concurrency Utilities for Java EE Provides managed executors, scheduled executors, thread factories and thread context propagation
    • Managed executors allow for scheduling and running of tasks in a Java EE managed environment
    • Interfaces in this specification extend from the familiar Java SE Concurrency APIs
    • Tasks can run with the thread context of the task submitter
    • A context service provides advanced capabilities to control thread context propagation
  • Java API for JSON Processing Provides JSR 353 JSON-P support. JSON-P is a standardized API library for the processing (parsing, generating, searching, and modifying) of JavaScript Object Notation (JSON).
    • The JSON-P APIs include streaming support for producing and consuming JSON data.
    • The JSON-P APIs allow the building of a Java object model to represent JSON text.
    • Why use JSON instead of XML?
  • Utilities for Remote Liberty Profile server WebSphere Developer Tools supports developing and deploying applications to a Liberty profile running on a remote machine. The following utilities are supported for the remote server:
    • Create SSL Certificate
    • Package Server
    • Create Collective Controller
    • Join Collective
    • Generate Dump for Support
  • Remote development tools for Liberty servers WebSphere Developer Tools (WDT) now allows you to develop with a Liberty server running on a remote system. Now you can connect, control and publish applications to remote servers!
    • You can edit server configuration files, publish applications, view and control server status, view and control application status, debug remote applications, run server utilities and view server logs all without having to leave the workbench:

These technologies are fully supported. In addition to these programming model features and tools we are also releasing support for:

  • Automatically scale your Liberty profile server environment based on workload
    • Auto-scaling starts and stops Liberty servers in the Liberty clusters automatically based on scaling policies. Dynamic routing routes HTTP requests automatically to the active Liberty servers in the Liberty cluster.   • OpenID Connect 1.0 provider and relying party OpenID Connect 1.0 is a simple identity protocol over OAuth 2.0.
    • Using OpenID Connect 1.0, an enterprise service provider can request user identity from an OpenID provider.
    • Liberty profile server could be configured to function as an OpenID Connect Provider, or to function as an OpenID Connect Client and Relying Party.
  • OpenID 2.0 relying party OpenID is a user controlled web identity, which allows user to use an existing account to sign in to multiple websites.
    • As OpenID relying party, WebSphere Application Server can be integrated with OpenID enabled social networking sites.

And what about the Migration Toolkit tech previews?

  • Migration Toolkit for Application Binaries (Tech Preview) Scans your application binaries without accessing the source code to identify the Java EE programming models used and to recommend the right-fit WebSphere Application Server edition (full profile or Liberty profile).

    • A command-line tool which enables developers or administrators to evaluate applications in minutes without accessing the source code.
    • Generates a visual report.
    • Does not require Eclipse so it’s perfect both for administrators and for developers using other IDEs.
