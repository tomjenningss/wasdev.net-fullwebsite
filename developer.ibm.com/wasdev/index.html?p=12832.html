Interceptors are classes that intercept method invocation or lifecycle events of a target class to perform some form of pre- or post-processing that is usually separate from the business logic, such as logging or auditing. The basic functionality of interceptors is defined in the interceptors specification and is extended in the CDI specification. This mini-series on interceptors shows how to create a simple interceptor and associate it with a CDI-managed bean. It also describes how to use multiple interceptors and define their ordering, the different life cycle events that can be intercepted, and how to globally enable an interceptor so it can be used across the whole application.

This article introduces you to writing applications with CDI interceptors by showing how to associate an interceptor with a CDI bean using interceptor bindings. The application passes a message from a servlet to the CDI bean, which adds to the message and returns it to the server. The interceptor also adds to the message, both before it is received by the bean and after it is returned.

Getting started

To build this application I used the Eclipse IDE for Java EE and Liberty Profile v9 Beta with Java EE 7 Technologies.

Building the application

Create a Dynamic Web Project

  1. Click File > New > Dynamic Web Project.
  2. Name it Interceptors.
  3. Uncheck Add project to an EAR.
  4. Create a new target runtime:
    1. Select WebSphere Application Server Liberty Profile.
    2. Click Next.
    3. Set the path to be your Liberty Runtime installation directory (ensure the runtime is at least V9).
    4. Click Finish.
  5. Click Finish.

Create an interceptor binding annotation

This is used to bind our interceptor class to the target bean.

  1. Right-click Interceptors.
  2. Click New > Annotation .
  3. Set the package name to com.wasdev.interceptors.interceptors.
  4. Set the name to InterceptMessage.
  5. Click Finish.
  6. Add the following code to InterceptMessage: 
    package com.wasdev.interceptors.interceptors;

import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

import javax.interceptor.InterceptorBinding;

@InterceptorBinding
@Target({ ElementType.TYPE, ElementType.CONSTRUCTOR })
@Retention(RetentionPolicy.RUNTIME)
public @interface InterceptMessage {}

Create an interceptor class

Only one method in the interceptor class can be designated to intercept method invocations from other classes and this must be annotated @AroundInvoke. This method will then intercept all method invocations within a target class that is associated with this interceptor. The intercept method shown below accesses the parameters being passed into the target method, alters the parameters and then passes the altered parameters on to the target method to be executed. The result from the target method is passed back to the intercept method, which again alters the value before returning it to the place from which it was originally called.

  1. Right-click Interceptors.
  2. Click New > Class.
  3. Set the package name to: com.wasdev.interceptors.interceptors.
  4. Set the name to: MessageInterceptor.
  5. Click Finish.
  6. Add the following code to MessageInterceptor:
package com.wasdev.interceptors.interceptors;

import javax.interceptor.AroundInvoke;
import javax.interceptor.Interceptor;
import javax.interceptor.InvocationContext;

@InterceptMessage
@Interceptor
public class MessageInterceptor {
	
	@AroundInvoke
    public Object intercept(InvocationContext ctx) throws Exception {
        Object[] params = ctx.getParameters();
        params[0] = params[0] + ", This message has been intercepted before the bean method was called";
        ctx.setParameters(params);
        Object message = ctx.proceed();
        return message + ", This message has been intercepted after the bean method was called";
    }

}

Create a CDI bean

This will have its methods intercepted by the Interceptor class.

  1. Right-click Interceptors.
  2. Click New > Class.
  3. Set the package name to com.wasdev.interceptors.beans.
  4. Set the name to SimpleBean.
  5. Click Finish.
  6. Add the following code to SimpleBean: 
    package com.wasdev.interceptors.beans;

import javax.enterprise.context.RequestScoped;
import com.wasdev.interceptors.interceptors.InterceptMessage;

@InterceptMessage
@RequestScoped
public class SimpleBean {

    public String sendMessage(String message){
    	return message + ", The message has reached the bean";
    }
}

Create a beans.xml file

This will be used to enable the interceptor.

  1. Right-click Interceptors.
  2. Click New > File.
  3. Set the parent folder to: Interceptors/WebContent/WEB-INF.
  4. Set the name to beans.xml.
  5. Click Finish.
  6. Add the following code to beans.xml: 
    <?xml version="1.0" encoding="UTF-8"?>
<beans bean-discovery-mode="annotated" xmlns="http://xmlns.jcp.org/xml/ns/javaee" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/beans_1_1.xsd">
	<interceptors>
	<class>com.wasdev.interceptors.interceptors.MessageInterceptor</class>
	</interceptors>
</beans>

Create a servlet

This will be used to test our interceptor.

  1. Right-click Interceptors.
  2. Click New > Servlet.
  3. Set the package name to com.wasdev.interceptors.servlet.
  4. Set the name to: TestServlet.
  5. Click Finish.
  6. Add the following code to TestServlet:
package com.wasdev.interceptors.servlet;

import java.io.IOException;
import java.io.PrintWriter;

import javax.inject.Inject;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

import com.wasdev.interceptors.beans.SimpleBean;

@WebServlet("/")
public class TestServlet extends HttpServlet {
       
	@Inject
	SimpleBean bean;

	protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
		PrintWriter out = response.getWriter();
        String message = bean.sendMessage("This is the servlet");
        out.println("nThe message returned to the servlet is:n " + message);
	}
}

Testing the application

Create a server instance

  1. Go to the Servers view in Eclipse
  2. Right-click New > Server.
  3. Under Select the server type, select WebSphere Application Server Liberty Profile.
  4. Ensure the Server Runtime Environment is set to the runtime environment created earlier.
  5. Click Finish.
  6. Expand the server we just created and double-click on Server Configuration.
  7. Go to the Design view by selecting the Design tab in the bottom left corner.
  8. Select Feature Manager and ensure that cdi-1.2 and servlet-3.1 are selected as features.
    (If the cdi-1.2 or servlet-3.1 features are not visible then you will need to ensure that you are using at least V9 of the Liberty Profile Runtime and WebSphere Developer Tools).


Run the application

  1. Right-click Interceptors.
  2. Click Run As > Run On Server.
  3. Under How do you want to select the server?, select Choose an existing server.
  4. Select the server we just created.
  5. Click Finish.
  6. A web browser should open and you should be able to see the following message:

    The message returned to the servlet is: This is the servlet, This message has been intercepted before the bean method was called, The message has reached the bean, This message has been intercepted after the bean method was called
